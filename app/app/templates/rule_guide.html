Rule Writing Guide

Use Detection Rules and Network Rules to validate ATT&CK techniques against the current analysis. A technique is treated as Validated when the PCAP evidence matches a rule that maps to that ATT&CK ID.

Rule Fields

FieldPurposeExample
NameHuman-readable rule title.Possible SMB Lateral Movement
DescriptionWhat the rule is trying to prove.SMB traffic to administrative services after authentication activity.
Event TypeThe normalized event type to match.flow, dns, icmp, http, tls
ProtocolProtocol or service name to match.SMB, DNS, HTTP, SSH, RDP
PortDestination or service port to match.445, 53, 22, 3389
SeverityFinding priority if the rule matches.medium, high, critical
ATT&CK IDsComma-separated techniques validated by the rule.T1021.002,T1046

How Rules Validate ATT&CK Coverage

  1. The parser turns packets into normalized events and flows.
  2. Rules are evaluated against those events.
  3. If a rule matches and includes ATT&CK IDs, those techniques are marked as rule-supported.
  4. If the technique is also observed in the PCAP and telemetry coverage exists, the heat map can classify it as Validated.

Tips for Better Rules

  • Prefer specific protocols and ports when possible.
  • Map rules only to techniques that the evidence actually supports.
  • Use higher severity for behavior that is unusual in your environment.
  • Keep broad rules informational or medium severity to avoid noisy results.
  • Use the Communications page to confirm which flows matched your rule.
