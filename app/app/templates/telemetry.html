Plugin Authoring Guide

A telemetry plugin defines how PCAP Mapper recognizes a product or log source and which ATT&CK-facing telemetry capabilities it contributes.

{
  "name": "Example EDR",
  "category": "EDR/XDR",
  "aliases": ["example edr", "example sensor"],
  "observed_components": ["Process Creation", "Network Connection Creation"],
  "potential_components": ["Registry Key Modification", "File Creation", "DNS Query"],
  "description": "Endpoint telemetry collected by Example EDR.",
  "confidence": "high",
  "enabled": true
}

Observed components should be evidence-backed capabilities. Potential components should describe capabilities the product can provide when fully configured. Use aliases that appear in logs, banners, filenames, normalized events, or vendor field names.
